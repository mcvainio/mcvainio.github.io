ArcticNet Basemaps Guidebook

 The goal of this collection is to provide a set of instructive examples which introduce a variety of open source plug-ins that come in handy when displaying geospatial data within the browser. Not all of these plug-ins were created specifically for web mapping, but we will be adapting them to suit our needs.

These examples were developed in the process of creating the ArcticNet Basemaps - a web portal maintained by the Ocean Mapping Group at UNB for the purpose of hosting the seabed & subsurface data collected by the CCGS Amundsen in the Canadian Arctic.

I aim to have all of my examples uploaded onto here by the end of Feb 2023. I am just working on writing my thesis at the moment...

Hosted at GitHub. Last updated: 31 Jan 2023.

2D Examples (Leaflet.js)

Leaflet Basics

Leaflet 101 example taken from their quickstart guide. Shows how to display a marker, a shape and a polygon.


Display Third-Party WMS

A Leaflet map with an inline HTML form that triggers an event handler to pass the user-inputted WMS URL into a variable. This variable is then called upon and displayed on the map.


Query Third-Party WMS

WMS used: Bathymetry data from the IHO.
Uses a click listener on a Leaflet map to grab a user-selected co-ordinate. Then, fires a single GetFeatureInfo query at the IHO's ESRI geoserver using that co-ord. The result (the depth property of the WMS) is displayed in an inline window on the map.


Query Third-Party WMS pt2

WMS used: EMOD bathymetry.
Same as above, but queries a general open geoserver rather than an ESRI-specific one. The previous example employed a plug-in designed specifically for handling ESRI requests. The current example employs jQuery instead to manually create an XMLHttpRequest.


Query Spatial Selection

Uses a rectangle on the map as the areal selection bounds (bbox) for performing multiple queries. The output results of those queries are stored in a JS array & assigned a variable. That array variable is then used as input to generate a histogram, which is displayed in a div to the side of the map using the Chart.JS plug-in.


Spatial Query Extended - Statistical Analysis

Performs an areal query using the methods outlined above. The results are displayed not only as a histogram, as various other statistical qualities & graphs related to the data found within the shape. Two different data layers from the map are compared, generating a QA/QC diagnostic window.


Sidebar GUI with Inline Layer Control

Illustrates how to modify the sidebar.js plugin in order to incorporate the layer control panel inline.



3D Examples (ThreeJS)

Plain Plane

Shows how to generate a wireframe mesh of a plane. Our subsequent examples will use this plane as a base, modifying it to model real-world terrain.


Modify Surface Elevations Using WCS

Generates a plane like the above example, but then loops through each vertex. For every point on the plane, the z value (the elevation) is changed into the corresponding value taken from a WCS.


Map WMS Texture onto Surface

Takes the wireframe from the above example & overlays an image texture. This texture is taken from a WMS that is clipped to the same geographic bounds as the plane & WCS data. Allow ~5s to load.


WMS Overlay Extended

Extends the previous example to include a toggle checkbox to switch the WMS overlay on/off. Also adds a dropdown menu for the user to select from multiple WMS overlay options. Adds a click listener to the 3D scene that will display the value of the WMS at the selected coordinate.


Modify Surface Colours Using ASCII JS Array

Rather than overlaying an image, we now try colouring our planar surface using a text array. In particular, we are using an ASCII file that corresponds to a sub-bottom dataset. The column that we've used to colour the pixels represents the intensity of the return at that depth. Essentially, this generates a cross-section.


3D Seabed (Topo and Subsurface)

Combining the WCS topo with WMS overlay, alongside the cross-sectional vertical curtain data that was generated from the ASCII example above.


Generate 3D Scene from 2D Map Selection

Starts with a 3D seabed model like the above example. This time though, there is a 2D Leaflet map alongside it. The overhead map lets the user change where the 3D model is generated from (ie. changes the bbox co-ords).



Final Web Map Portal GUI

Web Portal GUI

Extends the Leaflet sidebar.js add-on to incorporate spatial analysis & 3D modelling toolkits inline. This particular example was developed for UNB's Ocean Mapping Group in order to display seabed data that was collected aboard the CCGS Amundsen in the Canadian Arctic.
